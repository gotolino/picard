Building Picard From Source

See building HTSJDK from source.

Java version

Building the Picard command-line utilities requires Java 1.8. This can be done in one of the following ways:

  • Set an environment variable JAVA8_HOME that points to your Java 1.8 installation.
  • Pass to ant the argument -Djava8.home=path, where path is the directory of your Java 1.8 installation.
export $JAVA_HOME=Java_8_install_path
If you do not have Java 1.8 installed on your build machine, you may supply the location of you Java 1.8 installation instead, but there may be problems running code compiled this way on Java 1.8 JVMs.

Building Picard command-line utilities

To build the command-line utilities, change to the root of the Picard source tree, and run the following command:

ant -lib lib/ant package-commands

This will create an executable jar called picard.jar in the dist subdirectory. The format for invoking command-line utilities is as follows:

java jvm-args -jar dist/picard.jar PicardCommandName OPTION1=value1 OPTION2=value2...
See this page for additional details using this jar.

GA4GH API

It is possible to build a version of Picard that supports reading from the GA4GH API, e.g. Google Genomics:

  1. Fetch gatk-tools-java library from https://github.com/iliat/gatk-tools-java

  2. Build:

    ant gatk-tools-java-picard-jar

  3. Copy the resulting jar to lib/gatk-tools-java/ under the Picard folder, e.g.

    cp dist/gatk-tools-java-picard-1.0.jar ../picard/lib/gatk-tools-java/

  4. Build Picard:

    ant -lib lib/ant package-commands-ga4gh

  5. You can now run:

    java -jar dist/picard.jar ViewSam \
                INPUT=https://www.googleapis.com/genomics/v1beta2/readgroupsets/CK256frpGBD44IWHwLP22R4/ \
                GA4GH_CLIENT_SECRETS=../client_secrets.json

A set of Java command line tools for manipulating high-throughput sequencing data (HTS) data and formats.

Picard is implemented using the HTSJDK Java library HTSJDK, supporting accessing of common file formats, such as SAM and VCF, used for high-throughput sequencing data.

Introduction

SAM (Sequence Alignment/Map) format is a generic format for storing large nucleotide sequence alignments. SAM and related file formats are described in the hts-specs page.

Picard comprises Java-based command-line utilities that manipulate SAM files, and a Java API (HTSJDK) for creating new programs that read and write SAM files. Both SAM text format and SAM binary (BAM) format are supported.

To get help using and developing Picard, please use the GATK Forums.

Getting started: The Picard command-line tools are packaged as a single executable jar file. You can download the executable jar file by clicking the button at the top of the page labeled "Latest Release". For those who would just like to obtain the executable file to use it to analyze their data, just download the folder labeled picard-tools-2.x.x.zip. Open the downloaded file and place the folder containing the jar file in a convenient directory on your hard drive (or server).

For the tools to run properly, you must have Java 1.8 installed. It is recommended that you download the latest version of the Java Development Kit (JDK) and not the Java Runtime Environment. The JDK package can be found here: 

http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html
After installation, you need to verify that java has been installed properly. You can check your java version by opening your terminal and typing: 
java -version
If the output indicates version "1.8", you are good to go.

The next step entails navigating to your jar file. You can use the change directory (cd) command in your terminal to navigate to the directory where you placed the folder containing the jar file. Once there, you can type the following command: 

java -jar picard.jar -h
You should see a complete list of all the tools in the Picard program.

The format for invoking a specific tool is as follows: 

java jvm-args -jar picard.jar PicardToolName OPTION1=value1 OPTION2=value2...

Most of the commands are designed to run in 2GB of JVM, so the 

jvm-args
of "Xmx2g" is recommended.

The values of each "OPTION" or arguments can be files, strings, or an integer. For example, for an argument such as "INPUT=input.bam", where "INPUT" is the option and "input.bam" is the file. Another type of "OPTION" can be a string; "STRANDSPECIFICITY=NONE" where "STRANDSPECIFICITY" can be one of several choices including: {NONE, FIRST_READ_TRANSCRIPTION_STRAND, SECOND_READ_TRANSCRIPTION_STRAND}. For most string options, a default value is automatically invoked but can be changed by the user if required. Some arguments call for an integer, such as MINIMUM_LENGTH="Integer" where "Integer" is a numerical value such as 500.

Please note that most files used by the program as "INPUTS" and "OUTPUTS", require a file path e.g. 
INPUT=/my_data/input.bam
where the path is "/my_data/" and "/input.bam" is the file name.

A complete example for using the CollectAlignmentMetrics tools is as follows: 

java -jar picard.jar CollectAlignmentMetrics \

                   R=/my_data/reference.fasta \

                   I=/my_data/input.bam \

                   O=/my_data/output.txt

Picard is hosted by Github.com. The source code is available at the Picard project page here.

You can check out the latest source code with:

git clone git@github.com:broadinstitute/picard.git

The source code for HTSJDK is available on GitHub here. You can check out the latest source code with: 

git clone git@github.com:samtools/htsjdk.git

Enjoy!

Quick Start

See Building and Overview for details.

export $JAVA_HOME=Java_6_or_7_install_path
ant -lib lib/ant package-commands
java jvm-args -jar dist/picard.jar PicardCommandName OPTION1=value1 OPTION2=value2...

Getting Help

Picard is supported through the GATK Forums. Join the GATK Forum to get support for Picard programs among other tools supported by the Broad Institute.

Before Asking For Help

Before posting to the Forum, please do the following:

  • Try the latest version of Picard
  • See if your problem is discussed in the Frequently Asked Questions
  • Search the GATK Forums to see if your problem was previously discussed.
  • Run PicardValidateSamFile MODE=SUMMARY. Attempt to resolve or at least understand any problems reported.

When sending email about a problem, please include the following:

  • Command line(s) you used.
  • Program console output and metrics files. Repetitive console output may be abbreviated.
  • Entire stack trace if one was produced.
  • Version of JVM are you using (obtained by running 'java -version').
